# ソロプレナー実践ロードマップ
## 個人開発で月3万円（約200ドル）の自動収入を目指す9ステップ

---

### Step 1. アイデアの発掘：自分の「痛み」から探す

- **実践内容**: 日常生活や業務で感じる小さな「イライラ（ペインポイント）」をすべてメモする
- **ポイント**: 自分が何度も同じ不便を感じるものは、他の人も同様に困っている可能性が高く、作る価値がある
- **実例**: サムネイル作成が面倒という「痛み」から、プロンプト一つで生成できる「サムネAI」が誕生

---

### Step 2. 市場・競合調査：ニッチに尖らせる

- **実践内容**: Google検索やAI（Deep Research機能など）を使い、競合を調べる
- **ポイント**: 大手（CanvaやPhotoshopなど）が提供している機能であっても、**「1つの機能に特化（尖らせる）」**ことで個人でも勝機がある

---

### Step 3. MVP開発：バイブコーディングで最小限を作る

- **実践内容**: CursorやClaude Codeなどのツールを活用し、1つの痛みを解決する最小限の機能（MVP）を1週間程度で構築する

**推奨技術スタック：**

| 用途 | ツール | 備考 |
|---|---|---|
| フロント/バックエンド | Next.js | |
| データベース | Supabase / Turso | Tursoは100枠まで無料 |
| 認証 | Google認証 | 利便性重視 |
| 決済 | Stripe | |
| ストレージ | Cloudflare R2 | |

---

### Step 4. リリース：完璧主義を捨てる

- **実践内容**: 準備ができたら即リリースし、SNS（主にX）で告知する
- **必須設定**:
  - **Googleアナリティクス** — ユーザーの動きを追う
  - **サーチコンソール** — SEO対策の基盤

---

### Step 5. 分析：バケツの穴を見つける

- **実践内容**: データに基づき、ユーザーがどこで離脱しているか（バケツの穴）を特定する（ファネル分析）
- **効率化**: MCPサーバーを利用してAIにデータを分析させ、改善の仮説を立てさせるとスムーズ

---

### Step 6. 改善（PDCA）：ABテストの繰り返し

- **実践内容**: 登録率・課金率を高めるための施策を打つ
  - ログイン前にアプリを試せるUX設計にする
  - 課金ページのマイクロコピーを変更する
- **効果の目安**: 課金転換率が2%→4%に上がるだけで売上は2倍になる

---

### Step 7. PMF（プロダクトマーケットフィット）の達成

- **指標**: 新規ユーザーの**課金率が4〜6%**に達すれば、市場に受け入れられたと判断する
- **目指す状態**: ユーザーから「このアプリがなくなると困る」と思われること

---

### Step 8. ユニットエコノミクスの計算：赤字回避

- **計算式**: `LTV（顧客生涯価値） > CAC（顧客獲得単価）` であることを確認する
- **注意点**: AIのAPI利用料などのコストも差し引いて、利益が出るか必ず計算する

---

### Step 9. スケール：集客の拡大

**CPA（顧客獲得単価）0円の集客を優先する：**

| 手法 | 内容 |
|---|---|
| SEO / ASO | 検索順位の最適化 |
| SNS発信 | Xで開発過程を発信、TikTokでバズ狙い |
| リファラル | 紹介してくれたユーザーにクレジットを付与する仕組みを導入 |

---

## 月3万円達成の目安

| 指標 | 数値 |
|---|---|
| 月額料金（例） | 980円 |
| 必要有料ユーザー数 | 約31人 |
| 目標課金転換率 | 4〜6% |
| 必要無料ユーザー数（4%換算） | 約800人 |
